# Дорожная карта / Roadmap

Дорожная карта фиксирует ближайшие практические шаги. Она не обещает готовых результатов, а помогает координировать открытые задачи фонда.

## Этап 1. Публичная база

| Задача | Статус | Результат |
| --- | --- | --- |
| Создать структуру GitHub-репозитория | in progress | README, docs, research, data, outreach, project-management |
| Отделить публичные материалы от внутренних | in progress | Правила безопасности и проверки |
| Подготовить шаблоны issues и PR | in progress | Единый вход для задач |
| Описать миссию и направления | in progress | Документы для партнеров и участников |

## Этап 2. Исследования и данные

- Сформировать первичный реестр открытых источников данных.
- Описать исследовательские вопросы по биоразнообразию, климату, загрязнению и инфраструктуре данных.
- Подготовить первый воспроизводимый notebook без приватных данных.
- Определить правила цитирования источников и лицензий.

## Этап 3. Партнерства и события

- Подготовить список целевых организаций.
- Описать форматы сотрудничества для университетов, музеев, конференций и фондов.
- Создать первые письма и скрипты коммуникации.
- Подготовить короткие версии презентаций для партнеров и мероприятий.

## Этап 4. Публичный контур

- Настроить GitHub Discussions.
- Подготовить GitHub Pages как витрину документации.
- Добавить темы репозитория и описание.
- Сделать первый публичный release после проверки материалов.

## Контроль качества

- Каждое утверждение о партнерстве, данных или статусе проекта должно иметь источник.
- Все черновики помечаются как draft или needs verification.
- Данные с персональной, финансовой или закрытой информацией не публикуются.
