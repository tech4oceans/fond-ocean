# Visual Publication Standard

Ocean Fund treats preview images as a required part of public publication quality.

## Rule

Every public article, essay, research-facing note, one-pager, or flagship knowledge page should include a small thematic preview image that reflects the actual meaning of the material.

## Why

Preview images are not decoration. They help:

- strengthen visual identity;
- improve readability and public perception;
- make materials more legible in GitHub, Pages, and external sharing;
- connect the publication to its real theme through color, mood, and framing.

## Requirements

- the preview must be thematic, not generic;
- it must correspond to the meaning of the text;
- it must stay public-safe and not imply unverified claims;
- it should be reusable across language versions of the same material when appropriate;
- it should be stored in `assets/article-previews/` or another clearly named public asset directory.

## Current implementation

- article previews live in [assets/article-previews](../assets/article-previews/README.md)
- article publications live in [docs/articles](articles/index.md)

## Editorial rule

If a public material has no thematic preview image, it should be treated as visually incomplete until one is added.
