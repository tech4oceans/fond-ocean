# GitHub Discussions And Triage

Этот документ нужен как рабочая схема для Discussions, labels и стартовой модерации в `tech4oceans/fund-ocean`.

## Discussion Entry Points

### Announcements

Использовать для:

- новых публичных материалов;
- milestone updates;
- launch notes;
- новых dataset cards, pages или decks.

Pinned starter post:

`Welcome to Ocean Fund on GitHub`

Смысл:

- коротко объяснить миссию;
- показать, куда идти за research, data, outreach и partnership context;
- напомнить, что private contacts и confidential details не публикуются.

### Ideas

Использовать для:

- open suggestions;
- идеи по research, data, education, tooling;
- предложения по public website и repository structure.

Pinned starter post:

`Open idea intake: where contributors can help first`

### Q&A

Использовать для:

- вопросы по структуре репозитория;
- вопросы о contribution flow;
- уточнение, где лучше открыть issue, а где discussion.

### Partnerships

Целевая category: `Partnerships`.

Если отдельной category пока нет, временный маршрут такой:

- использовать `General` для одного pinned discussion вида `Partnership intake: start here with public information only`;
- после появления category `Partnerships` перенести этот discussion туда или создать новый.

## Starter Discussion Copy

### Welcome / announcement

Title:

`Welcome to Ocean Fund on GitHub`

Body:

```md
Ocean Fund is building an open hub for ocean research, marine data, education, public knowledge, and partnerships.

If you are new here:

- start with the README;
- read the mission and one research or data page;
- use Ideas for open suggestions;
- use Q&A for orientation questions;
- use the partner entry materials if you represent an organization.

Please keep discussions public-safe:

- no private phone numbers or personal emails;
- no private documents or confidential terms;
- no unverified partnership claims;
- no tokens, credentials, or internal system links.
```

### Ideas intake

Title:

`Open idea intake: where contributors can help first`

Body:

```md
Use this discussion to suggest one practical idea that could improve Ocean Fund.

Strong ideas usually fit one of these tracks:

- research directions;
- marine data sources;
- education and public literacy;
- outreach and events;
- partnership models;
- public website and showcase;
- governance and repository clarity.

Please make the idea concrete and suggest the smallest next step.
```

### Partnership intake

Title:

`Partnership intake: start here with public information only`

Body:

```md
If you represent a university, museum, lab, nonprofit, conference, or public institution, this is the public-safe place to begin.

Please share only:

- organization name;
- public website;
- why collaboration is relevant;
- one realistic format for a first step;
- one possible public-facing outcome.

Do not post:

- personal phone numbers;
- personal email addresses;
- private documents;
- confidential terms;
- unconfirmed statements that a partnership already exists.
```

- `research`
- `data`
- `outreach`
- `partnership`
- `brand`
- `website`
- `governance`
- `content`
- `legal`
- `priority-high`
- `needs-triage`
- `good first issue`
- `help wanted`

Recommended additions:

- `needs-verification` for unconfirmed facts, sources, or claims
- `public-safe` for items already reviewed for safe publication
- `blocked` for tasks waiting on access, approval, or an external dependency

## Triage Rules

### Open a discussion when

- the topic is exploratory;
- the answer is not yet a concrete task;
- the user needs orientation;
- the topic is partner-facing but still early and public-safe.

### Open an issue when

- there is a concrete deliverable;
- the task can be assigned or tracked;
- the work has a clear next step;
- the discussion already produced an action item.

### Escalate outside GitHub when

- personal data appears;
- confidential documents are needed;
- funding or legal claims need approval;
- the conversation requires a private channel.
