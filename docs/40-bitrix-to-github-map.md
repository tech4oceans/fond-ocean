# Карта переноса Bitrix24 -> GitHub

## Источник

Рабочая группа Bitrix24: `ОКЕАН`.
Хранилище группы используется только как внутренний источник материалов и не раскрывается на уровне технических идентификаторов.

## Уже индексируется

- корень диска группы;
- ключевые папки;
- задачи группы;
- задачи по словам ОКЕАН/Ocean/tech4oceans;
- digest последних сообщений чата группы.

## Кандидаты на перенос в GitHub

- УСТАВНЫЕ: только публичные реквизиты и ссылки/описания документов, не сами сканы без отдельного решения.
- ПУБЛИКАЦИИ: миссия, платформа, эссе, климат, лоцманское обеспечение.
- ИНДЕКСЫ: описания индексов, методики, перечни.
- РЕЙТИНГИ: общая статья и таблицы, если нет персональных данных.
- ПИСЬМА: шаблоны обращений, обезличенные инструкции, публичные версии писем.
- Материалы о представителях фонда: только утвержденные публичные тезисы без персональных документов.

## Не переносить

- папку ДОСТУПЫ;
- полные чаты;
- личные документы;
- файлы с паспортными, контактными или банковскими данными;
- любые ключи, токены, webhook URL, пароли;
- неподтвержденные «внутренние» договоренности как публичные факты.

## Нормализация материалов

Каждый переносимый материал должен пройти четыре шага:

1. Очистка от персональных данных и случайных комментариев.
2. Пометка статуса: `draft`, `needs verification`, `public ready`.
3. Привязка к направлению фонда.
4. Ссылка на исходный объект Bitrix без скачивания файла в репозиторий.
