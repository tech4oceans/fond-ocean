# خريطة الطريق

![roadmap preview](../../assets/docs-previews/ar/roadmap.svg)


وتحدد خارطة الطريق الخطوات العملية التالية. إنه لا يعد بنتائج جاهزة، ولكنه يساعد على تنسيق المهام المفتوحة للمؤسسة.

## المرحلة 1. القاعدة العامة

| مهمة | حالة | نتيجة |
| --- | --- | --- |
| قم بإنشاء هيكل مستودع GitHub | في تَقَدم | README، المستندات، البحث، البيانات، التوعية، إدارة المشاريع |
| فصل المواد العامة عن الداخلية | في تَقَدم | قواعد السلامة والتفتيش |
| إعداد القضايا وقوالب العلاقات العامة | في تَقَدم | تسجيل الدخول الموحد للمهام |
| وصف المهمة والاتجاهات | في تَقَدم | وثائق للشركاء والمشاركين |

## المرحلة الثانية: البحوث والبيانات

- إنشاء سجل أساسي لمصادر البيانات المفتوحة.
- وصف الأسئلة البحثية المتعلقة بالتنوع البيولوجي والمناخ والتلوث والبنية التحتية للبيانات.
- قم بإعداد أول كمبيوتر محمول قابل للتشغيل بدون بيانات خاصة.
- تحديد قواعد الاستشهاد بالمصادر والتراخيص.

## المرحلة الثالثة: الشراكات والفعاليات

- إعداد قائمة بالمنظمات المستهدفة.
- وصف أشكال التعاون للجامعات والمتاحف والمؤتمرات والمؤسسات.
- قم بإنشاء الحروف الأولى ونصوص الاتصال.
- قم بإعداد نسخ قصيرة من العروض التقديمية للشركاء والأحداث.

## المرحلة 4. المخطط العام

- قم بإعداد مناقشات جيثب.
- قم بإعداد صفحات GitHub كعرض للتوثيق.
- إضافة موضوعات المستودع والوصف.
- قم بإعداد الإصدار العام الأول بعد التحقق من المواد.

## ضبط الجودة

- يجب أن يكون لكل بيان حول شراكة أو بيانات أو حالة مشروع مصدر.
- يتم وضع علامة على جميع المسودات كمسودة أو تحتاج إلى التحقق.
- لا يتم نشر البيانات التي تحتوي على معلومات شخصية أو مالية أو حساسة.
