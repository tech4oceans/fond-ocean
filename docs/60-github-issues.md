# GitHub Issues

Это стартовый публичный набор задач для витрины Ocean Fund на GitHub. Он должен показывать движение проекта и сразу объяснять новым участникам, где можно помочь.

Статус на `2026-05-12`: первые 6 issues уже созданы в `tech4oceans/fund-ocean`. Ниже зафиксирован канонический набор из 10 стартовых задач, который можно поддерживать как публичную очередь запуска.

## 1. Research - собрать 10 приоритетных тем по океану и климату

Labels: `research`, `content`, `priority-high`

Собрать и структурировать 10 направлений, которые задают первые исследовательские треки Ocean Fund: океан и климат, биоразнообразие, загрязнение, батиметрия, marine monitoring, ocean literacy и смежные темы. Для каждой темы нужен краткий смысл, возможный результат и список базовых источников.

Критерий готовности:

- список из 10 тем опубликован в публично пригодной форме;
- у каждой темы есть краткое описание и practical next step;
- ссылки на mission и research directions согласованы.

## 2. Data - оформить 5 проверенных открытых источников данных

Labels: `data`, `research`, `priority-high`

Подготовить 5 dataset cards для надежных открытых источников, например Copernicus Marine, OBIS, GEBCO, EMODnet, NOAA/IOOS. Для каждого источника указать ссылку, тип данных, лицензию, дату доступа, ограничения и потенциальную ценность для фонда.

Критерий готовности:

- оформлено 5 dataset cards;
- у каждого источника есть license/access notes;
- нет непроверенных claims о доступности и качестве данных.

## 3. Outreach - подготовить короткое письмо для университетов и музеев

Labels: `outreach`, `partnership`, `content`

Собрать краткое английское и русское письмо для первого публичного контакта с университетами, музеями и научными центрами. Письмо должно объяснять, кто такой Ocean Fund, зачем нужен open project hub и какой формат первого разговора предлагается.

Критерий готовности:

- есть RU и EN версии;
- письмо не обещает неподтвержденных партнерств;
- текст согласован с публичной миссией.

## 4. Brand - утвердить англоязычное написание названия и описание

Labels: `brand`, `content`, `priority-high`

Утвердить официальное английское написание названия, короткое one-line описание для GitHub profile и repository description, а также единый словарь формулировок для README, social preview и profile README.

Критерий готовности:

- зафиксировано одно RU/EN naming pair;
- есть repository description и profile bio;
- словарь формулировок совпадает в ключевых публичных точках.

## 5. Website - привести `public/` к единой публичной версии

Labels: `website`, `content`, `needs-triage`

Подготовить стартовую public-витрину для GitHub Pages или внешнего сайта. Минимум: первый экран, краткая миссия, направления, способ участия, ссылка на GitHub hub и безопасные публичные контакты после утверждения.

Критерий готовности:

- `public/` содержит связную public-safe структуру;
- есть landing copy и partner entry;
- репозиторий готов к первому запуску Pages или внешней витрины.

## 6. Governance - определить публичные контакты и лицензионную стратегию

Labels: `governance`, `legal`, `priority-high`

Определить, какие email, ссылки, юридические формулировки и лицензии могут быть опубликованы в GitHub-профиле и в репозитории. Не публиковать персональные данные, сканы документов и финансовые реквизиты до отдельного утверждения.

Критерий готовности:

- определены допустимые публичные контакты;
- зафиксирована временная или финальная лицензионная позиция;
- checklist по safe publication согласован.

## 7. GitHub setup - завершить About, homepage и social preview

Labels: `brand`, `website`, `priority-high`

Проверить, что GitHub-репозиторий оформлен как публичная витрина: description, homepage, topics, social preview image, social framing и навигация между README, profile и `public/`.

Критерий готовности:

- homepage больше не пустой;
- social preview загружен;
- About совпадает с README и profile copy.

## 8. Discussions - утвердить moderation flow и partner intake

Labels: `governance`, `partnership`, `needs-triage`

Привести Discussions к рабочему виду: определить стартовые pinned discussions, правила модерации, маршрутизацию вопросов и идею открытого partner intake без публикации персональных данных.

Критерий готовности:

- есть welcome discussion;
- есть idea intake и partner intake entry point;
- описано, когда вести разговор в discussion, а когда переводить в issue.

## 9. Partnerships - собрать первую public-safe карту организаций

Labels: `partnership`, `research`, `content`

Собрать 20-30 организаций для первого partner atlas без персональных контактов: университеты, музеи, научные центры, конференции, open-source communities и nonprofits, релевантные океану, климату, образованию и данным.

Критерий готовности:

- есть структурированный список организаций;
- все ссылки публичные;
- форматы сотрудничества описаны без неподтвержденных договоренностей.

## 10. Data showcase - выбрать первый демонстрационный dataset и результат

Labels: `data`, `website`, `needs-triage`

Выбрать один демонстрационный dataset и определить первый публичный результат: dataset card, notebook, карта, визуализация, partner brief или короткий showcase page для витрины Ocean Fund.

Критерий готовности:

- выбран один приоритетный dataset;
- зафиксирован тип первого результата;
- есть понятный next step для реализации.
