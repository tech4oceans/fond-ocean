# Security

Фонд «Океан» не публикует в этом репозитории секреты, приватные документы, персональные данные, банковские реквизиты или внутренние доступы.

## Что считается чувствительным

- API keys, tokens, passwords, webhook URL;
- персональные документы и частные контакты;
- финансовые реквизиты и платежные инструкции до отдельного утверждения;
- закрытые выгрузки из внутренних систем;
- сырые чаты и документы с персональными данными.

## Если вы нашли проблему

Не создавайте публичный issue с секретом или персональными данными.

Публичный security contact будет добавлен после утверждения фонда: `security@example.org`.

До появления утвержденного контакта сообщайте maintainer'ам через безопасный внутренний канал фонда.

## Для contributors

Перед pull request проверьте:

- нет ли секретов в тексте и истории изменений;
- не добавлены ли приватные файлы;
- не содержит ли материал неподтвержденных публичных заявлений;
- указаны ли лицензии и источники данных.
